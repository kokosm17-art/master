## 2. 개발 필요 항목 상세

### 2.1 🔴 [P0] 콘티 빌더 백엔드 연동 — 핵심 기능

현재 `storyboard-builder.tsx`의 폼 입력은 로컬 `useState`로만 관리되며, "콘티 완성하기" 버튼 클릭 시 CTA 섹션으로 스크롤만 합니다. 실제 콘티 생성 로직이 필요합니다.

#### 개발 항목

| # | 항목 | 설명 | 예상 난이도 |
|---|------|------|------------|
| 2.1.1 | **AI 콘티 생성 API** | 사용자가 입력한 10개 필드(brandName, mainProblem, solution, targetAudience, benefit1~3, certifications, reviews, socialProof)를 기반으로 상세페이지 콘티를 자동 생성하는 API | 🔴 높음 |
| 2.1.2 | **프롬프트 엔지니어링** | 5단계 성공 프레임워크 (Hook → Agitate → Solution → Social Proof → Trust & CTA) 로직을 AI 프롬프트에 반영 | 🔴 높음 |
| 2.1.3 | **콘티 결과 페이지** | 생성된 콘티를 보여주는 전용 결과 페이지 (현재 없음) | 🟡 중간 |
| 2.1.4 | **콘티 PDF/이미지 내보내기** | 완성된 콘티를 PDF 또는 이미지로 다운로드 | 🟡 중간 |
| 2.1.5 | **폼 유효성 검사 강화** | 현재 최소한의 검증만 존재 (brandName 또는 mainProblem이 비어있지 않으면 다음 단계로 이동 가능). 각 필드별 상세 유효성 검사 및 에러 메시지 필요 | 🟢 낮음 |
| 2.1.6 | **로딩/생성 중 상태 UI** | AI 생성 중 스켈레톤 또는 로딩 애니메이션 | 🟢 낮음 |

#### 기술 제안

```
입력 데이터 (FormData)
    ↓
Next.js API Route (/api/generate-storyboard)
    ↓
AI 서비스 (OpenAI GPT-4 / Claude API)
    ↓ 프롬프트: 5단계 프레임워크 기반 콘티 생성
    ↓
결과 (구조화된 JSON)
    ↓
콘티 결과 페이지 렌더링 + 내보내기
```

#### 제안 DB 스키마 (개념)

```
Users
├── id (PK)
├── email
├── name
├── provider (kakao / naver / google / email)
├── created_at
└── updated_at

Storyboards
├── id (PK)
├── user_id (FK → Users)
├── brand_name
├── main_problem
├── solution
├── target_audience
├── benefits (JSON)
├── certifications
├── reviews
├── social_proof
├── generated_content (JSON) ← AI가 생성한 콘티 결과
├── status (draft / completed / exported)
├── created_at
└── updated_at

Payments
├── id (PK)
├── user_id (FK → Users)
├── storyboard_id (FK → Storyboards)
├── amount
├── payment_method
├── payment_status
├── pg_transaction_id
├── created_at
└── updated_at
```

#### 기술 제안

## 5. 참고 사항

### 현재 코드베이스에서 주의할 점

1. **v0.app 생성 코드**: 이 프로젝트는 v0.app에서 생성된 프로토타입이므로, 프로덕션 전환 시 코드 품질 검토가 필요합니다.
2. **shadcn/ui 컴포넌트 57개**: `components/ui/` 하위에 57개의 UI 컴포넌트가 포함되어 있으나, 실제 사용되는 것은 일부(Button, Input, Textarea 등)입니다. 사용하지 않는 컴포넌트 정리를 권장합니다.
3. **Tailwind CSS v4**: 최신 Tailwind v4를 사용하고 있으며, oklch 색상 시스템을 채택하고 있습니다. 기존 방식과 다소 다르므로 팀 내 공유가 필요합니다.
4. **하드코딩된 데이터**: 통계 수치(400억, 1,000+, 98%), 타임라인 마일스톤 데이터 등이 모두 컴포넌트 파일 내에 하드코딩되어 있습니다. 관리 편의를 위해 별도 설정 파일 또는 CMS 연동을 고려해야 합니다.
5. **`theme-provider.tsx` 미사용**: 파일이 존재하나 `layout.tsx`에서 사용되지 않고 있습니다.
